# Task 6.1 - Configuração de Produção Hostinger

## Summary
Entregue a base de configuração de produção para a Hostinger (hospedagem compartilhada,
sem Redis/worker): modelo de `.env` de produção, `.htaccess` com Gzip + cache de assets,
e a documentação de SSL, document root, cron único do scheduler, fila no banco e build do
Vite. Tudo incorporando as lições do deploy de teste real já feito.

## Details
- **`.env.production.example`** criado a partir do `.env.example`, ajustado para produção:
  `APP_ENV=production`, `APP_DEBUG=false`, `APP_URL=https://...`, `LOG_LEVEL=warning`,
  `SESSION_DRIVER=database`, `QUEUE_CONNECTION=database`, `CACHE_STORE=file`, DB do MySQL da
  Hostinger e bloco `MAIL_*` SMTP (smtp.hostinger.com, 465/SSL). Removidas as chaves de
  Redis/Memcached/AWS irrelevantes para o alvo. Campos sensíveis marcados como `<< ... >>`
  (sem segredos reais). Decisão: arquivo nomeado `.env.production.example` para conviver com
  o `.env.example` de desenvolvimento sem sobrescrevê-lo.
- **`public/.htaccess`** ampliado de forma **aditiva** (sem mexer no bloco de rewrite do
  Laravel): `mod_deflate` (Gzip de HTML/CSS/JS/JSON/SVG/fontes), `mod_expires` (cache de 1 ano
  para os assets versionados do Vite, fontes e imagens) e `mod_headers` para **proibir cache**
  do Service Worker e do manifesto do PWA. Todos os blocos protegidos por `<IfModule>` — se o
  módulo não existir no servidor, são ignorados sem quebrar o site.
- **Documento `docs/01-configuracao-producao.md`** (PT-BR, leigo) cobrindo cada item de
  produção com o "porquê" e marcando os **passos do usuário** (banco MySQL, SSL Let's Encrypt,
  document root → `public`, cron, SMTP). Inclui o **checklist de produção** verificável.
- Cron único documentado: `* * * * * cd <caminho> && php artisan schedule:run` (a cada minuto),
  que dispara o job `RecalculateStreaks` (00:10) via fila no banco — confirmado em
  `routes/console.php` e no log da Task 4.1.
- Build de produção (Vite): recomendada a estratégia **buildar localmente e enviar `public/build`**
  (Node não confiável no shared hosting), conforme validado no deploy de teste.

## Important Findings
- **`php artisan storage:link` falha na Hostinger** (`exec()` desabilitada). Documentado:
  para o FlowFin é irrelevante (não serve uploads públicos) → **pode pular**.
- **`pdo_mysql`** precisa estar habilitado no PHP do host (em geral já vem ligado na Hostinger).
- Decisão para revisão do Manager: nome do arquivo de modelo é `.env.production.example`
  (não sobrescreve o `.env.example` de dev). Ativação de **SSL, cron e SMTP** são passos do
  usuário (credenciais/painel) — listados para o Manager pausar e orientar.
