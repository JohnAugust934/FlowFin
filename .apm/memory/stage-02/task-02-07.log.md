# Task 2.7 - Refinamento Visual Mobile & Navegação de Categorias

## Summary
Refinamento visual mobile do MVP em três rodadas: (1) linguagem de vidro fosco + tema claro/escuro/sistema + categorias no menu mobile; (2) itens do histórico arejados, ajuste do gráfico e ícone da marca no topo; (3) substituição da visualização de gastos por categoria (donut → ranking com barras de proporção), pois o usuário não gostou do donut mesmo após ajuste. Build compila, bundle JS reduzido, telas do MVP renderizam (200); resta validação visual guiada (status Partial).

## Details
### Rodada 1 — base (commit `1e928cd`)
- **Categorias no menu mobile:** `bottom-nav.blade.php` — "Metas" (placeholder) substituído por **Categorias** (`route('categories.manage')`, ícone + estado ativo).
- **Fundamento de design:** `tailwind.config.js` (`darkMode: 'class'`, sombras `glass`); `app.css` — `.app-canvas` (fundo ambiente), `.glass`, `.card` translúcido, `.glass-row`, variantes `dark:` para inputs e `.btn-secondary`.
- **Tema claro/escuro/sistema:** Alpine `themeControl` (persiste em `localStorage`, classe `dark`, acompanha `prefers-color-scheme`); script inline no `<head>` aplica antes da pintura (sem flash); `x-theme-toggle` acessível na navegação desktop e cabeçalho mobile.
- **Consistência:** vidro + `dark:` em dashboard, histórico, categorias, registro rápido e avisos.

### Rodada 2 — ajustes (commit `fff6eb8`)
- **Histórico mobile arejado:** itens com `p-4`, ícone em quadrado arredondado destacado, valor proeminente, data/classificação como metadados secundários (badge sutil); ações com toque confortável; filtros/paginação intactos.
- **Ícone da marca no topo:** `app.blade.php` e `navigation.blade.php` passam a `<x-brand-icon>`; logo com texto preservada só no `guest.blade.php` (login).
- (O donut foi ajustado nesta rodada, mas reprovado — ver rodada 3.)

### Rodada 3 — visualização de gastos por categoria (commit `3c82436`)
- **Pesquisa:** usei a skill `frontend-design` e pesquisa web de padrões mobile de "gastos por categoria". Conclusão: em celular, ranking/listagem com barras de proporção comunica melhor "para onde foi o dinheiro" do que um donut com legenda apertada (priorização da hierarquia).
- **Decisão (substituição do donut):** removi o gráfico de rosca Chart.js e implementei um **ranking de categorias**: barra de visão geral (composição empilhada por cor) com o total, e a lista ordenada (desc) onde cada categoria mostra **ícone+cor**, **nome**, **valor em R$** e **percentual do total**, com barra de proporção própria. Tudo em CSS — funciona nos dois temas com bom contraste, sem as limitações de legenda/tema do Chart.js.
- **Componente `dashboard`:** removi estado/lifecycle do chart (`chart`, `renderChart`, `$nextTick`, listener `theme-changed`); adicionei getters `totalSaidas`/`totalSaidasLabel`/`categoryBreakdown` (cor resolvida, `pct`, rótulos) e helper `categoryIcon`. Contrato de API inalterado (mesmo `by_category` de `GET /api/dashboard`), `centsToBRL` mantido, estado vazio preservado, reatividade a `transaction-saved` mantida (tema agora é puro CSS, dispensa redraw).
- **Performance:** Chart.js deixou de ser usado; removi o import em `app.js`. Bundle JS caiu de ~262 KB (90,5 KB gzip) para ~59 KB (21,2 KB gzip). `chart.js` segue listado no `package.json` (sem desinstalar — não-invasivo).

## Important Findings
- **Spec divergente (aprovado pelo usuário):** a Spec previa "gráfico de rosca"; o usuário priorizou o resultado visual e autorizou trocar a forma. A entrega usa **ranking com barras de proporção** em vez do donut. Vale o Manager registrar essa mudança de forma na Spec/Memory.
- **Chart.js órfão:** com a remoção do donut, `chart.js` continua como dependência em `package.json` mas não é mais importado/usado. Pode ser desinstalado numa limpeza futura se nenhuma outra tela for usar gráficos.
- **Superfícies fora do escopo do MVP sem tema escuro:** página de **Perfil** (Breeze) e **dropdown do usuário** no desktop seguem claras no modo escuro (legíveis, inconsistentes). Sugiro Task futura curta se o tema escuro virar recurso oficial.
- **Componentes compartilhados** (`.card`, `.btn-secondary`, `card.blade`, `toast.blade`, `transaction-form`) ficaram translúcidos/cientes do tema — retrocompatível, mas afeta o visual de qualquer tela que os reutilize.
