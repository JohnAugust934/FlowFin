## Important Findings
- **Sem MySQL nem `pdo_mysql` no ambiente local de desenvolvimento.** O PHP 8.5 CLI desta máquina expõe apenas os drivers PDO `pgsql` e `sqlite`; não há servidor MySQL rodando. Consequência prática: qualquer Task futura que valide código dependente de banco **não conseguirá rodar contra MySQL localmente** sem antes (a) habilitar a extensão `pdo_mysql` no `php.ini` e (b) subir um MySQL local (ou usar o SQLite como banco de validação descartável, como fiz aqui). A produção na Hostinger já inclui `pdo_mysql` por padrão, então o `.env` de referência em MySQL está correto para o alvo — apenas a validação local exige contorno.
- **`SESSION_DRIVER=database` (padrão do Laravel 13)**: torna toda renderização de página dependente do banco. Mantive o padrão por ser adequado à hospedagem compartilhada com MySQL; vale ciência para depurações locais (renderização exige DB acessível).
- **Migrations consolidadas**: `create_jobs_table` e `create_cache_table` já vêm por padrão no Laravel 13 — `queue:table`/`cache:table` são desnecessários nesta versão.

### Atualização pós-entrega — ambiente MySQL local resolvido
O usuário instalou o MySQL Server (serviço `MySQL80`, porta 3306). Nesta sessão eu: (a) habilitei `extension=pdo_mysql` em `C:\tools\php85\php.ini` (a DLL já existia em `ext/`); (b) criei o database `flowfin` (utf8mb4/unicode_ci) com as credenciais locais `root`/`root`; (c) defini `DB_PASSWORD=root` no `.env` local (gitignored — não afeta o repositório). Revalidei **com a configuração real, sem overrides**: `php artisan migrate` roda as 3 migrations no MySQL; `php artisan serve` retorna HTTP 200 em `/`, `/login`, `/register` (sessão em banco funcionando, sem o 500 anterior); e `/img/brand/logo_flowfin.svg` serve em `image/svg+xml`. **O achado importante deixa de ser bloqueio para o ambiente local** — Tasks futuras dependentes de banco já podem validar contra MySQL local. Observação para o ambiente de produção/outras máquinas: garantir `pdo_mysql` habilitado no PHP é pré-requisito.
