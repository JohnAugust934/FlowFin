# Task 1.3 - Modelo de Dados, Migrations & Seeders

## Summary
Schema completo do FlowFin implementado: 8 migrations (perfil de usuário estendido + 7 entidades), 8 models Eloquent com relacionamentos e soft delete, e seeder das 9 categorias pré-definidas. `migrate:fresh --seed` roda sem erro contra o MySQL local `flowfin`.

## Details
Trabalho no worktree `C:\_PROJETOS\FlowFin\.apm\worktrees\modelo-dados` (branch `feature/modelo-dados`). Bootstrap feito: `.env` copiado do diretório raiz e `composer install` executado (composer/php resolvidos via PowerShell em `C:\tools\`, não estavam no PATH do bash).

Decisões de modelagem:
- **Dinheiro:** todas as colunas monetárias são `bigInteger` (centavos) — `users.monthly_income`, `transactions.amount`, `budgets.monthly_limit`, `goals.target_amount`/`saved_amount`, `recurrences.amount`, `investments.amount`. Nenhum decimal/float. Casts `integer` nos models.
- **Soft delete:** `softDeletes()` em todas as entidades do usuário (categories, transactions, budgets, goals, recurrences, investments) + educational_contents; trait `SoftDeletes` nos models. Tabela `users` não recebeu soft delete (exclusão de conta é fluxo LGPD próprio).
- **Categorias:** `user_id` anulável — pré-definidas têm `user_id` nulo e `is_predefined=true`; personalizadas pertencem a um usuário (`cascadeOnDelete`).
- **Transações:** enums `type` (entrada/saida) e `classification` (necessidade/desejo, anulável p/ saídas); `is_recurring` e FK `recurrence_id` (nullOnDelete) ligando à conta fixa de origem. Índices em `user_id`, `category_id`, `date`, `type`.
- **FKs:** `user_id` sempre `cascadeOnDelete`; `category_id` em transactions/recurrences usa `nullOnDelete` (preserva histórico se categoria for removida); budgets usa `cascadeOnDelete` na categoria.
- **Ordem das migrations:** categories e recurrences antes de transactions para satisfazer as FKs.
- Seeder `CategorySeeder` usa `updateOrCreate` (idempotente) e foi registrado em `DatabaseSeeder` via `$this->call()`.
- Código formatado com Pint.

## Important Findings
- O model `User` do Breeze (Laravel 13) usa o **estilo de atributos PHP** novo (`#[Fillable([...])]` e `#[Hidden([...])]`) em vez das propriedades `$fillable`/`$hidden`. Estendi consistentemente: adicionei `monthly_income` ao atributo `#[Fillable]` e o cast `integer`. Workers que tocarem o `User` depois devem seguir esse estilo de atributos, não criar propriedades duplicadas.
- Os novos models referenciam classes de Factory nos docblocks (`@use HasFactory<...Factory>`) e o Pint adicionou os `use` correspondentes. Essas Factories **ainda não existem** (apenas `UserFactory`). É inofensivo (referência só em docblock, sem uso em runtime), mas quem for escrever feature tests precisará criar as Factories das entidades.
- `migrate:fresh` apaga e recria as tabelas do database `flowfin` compartilhado entre worktrees — relevante para coordenação se outro Worker depender do estado do banco.
