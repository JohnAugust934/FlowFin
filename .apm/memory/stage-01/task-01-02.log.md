## Compatibility Concerns
- **Componentes compartilhados do Breeze foram modificados** (botões, inputs, nav-links, labels) para adotar a identidade da marca. As telas de autenticação existentes foram verificadas e renderizam corretamente, mas qualquer Worker que dependa do estilo original do Breeze verá o novo visual da marca — comportamento desejado para consistência.
- **Toast depende de stack de scripts:** o componente `x-toast` usa `@push('scripts')` e os layouts (`app`, `guest`) incluem `@stack('scripts')` antes de `</body>`. Telas futuras devem usar `x-app-layout`/`x-guest-layout` para que toasts e flash de sessão funcionem. Disparo: `$dispatch('toast', { type, message })` ou `window.dispatchEvent(new CustomEvent('toast', {detail}))`.

## Important Findings
- **Bootstrap do worktree:** `composer` não está no PATH do Git Bash neste ambiente — foi necessário rodar `composer install` via PowerShell (`C:\tools\composer\composer.bat`). `npm` funciona no Git Bash. Relevante para qualquer Worker que opere em worktree isolado.
- **Tailwind v3 confirmado** apesar do `@tailwindcss/vite ^4` constar no `package.json` (não utilizado). Tokens devem ser configurados em `tailwind.config.js`, não via `@theme` no CSS.
- **Safelist no Tailwind:** as amostras de paleta da página de demo usam classes interpoladas (`bg-brand-{n}`) que o scanner do JIT não detecta; foi adicionada uma `safelist` por regex no `tailwind.config.js` para `(bg|text|border)-(brand|emerald|neutral)-(50..900)`. Telas de produto devem usar classes literais; a safelist existe sobretudo para a demo.
- **Fonte self-hosted** (`@fontsource/inter`) em vez de CDN — bundle offline. Decisão alinhada à restrição de hospedagem compartilhada e ao requisito de funcionar offline futuramente.
