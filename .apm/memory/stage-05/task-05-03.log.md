# Task 5.3 — Export CSV/PDF + LGPD

## Summary
Implementado o export do relatório mensal em **CSV** e **PDF**, o **export completo dos
dados** do usuário (LGPD — portabilidade) e a **exclusão definitiva da conta** (LGPD —
purge físico). Tudo escopado ao usuário autenticado, PT-BR, R$ no formato brasileiro e
datas dd/mm/aaaa na apresentação. Suíte completa verde (155/155).

## Details
- **Lib de PDF:** `barryvdh/laravel-dompdf` **v3.1.2** (DomPDF puro PHP — compatível com
  hospedagem compartilhada, sem Redis/worker). Instalada via `composer require` (PowerShell).
  CSV é **nativo** (`fputcsv` em stream) — sem dependência extra.
- **Relatório mensal (`MonthlyReportService` + `ReportExportController@monthly`):** reaproveita
  `DashboardService` para os totais (entrou/saiu/sobrou) e carrega as transações do mês com
  **eager loading** da categoria (`with('category:id,name')`, sem N+1). Valores mantidos em
  CENTAVOS internamente; formatados via `App\Support\Money` só na saída.
  - **CSV:** separador `;` (padrão do Excel pt-BR, já que a vírgula é decimal) + **BOM UTF-8**
    para acentos corretos no Excel; cabeçalho de resumo + linhas (Data, Tipo, Categoria,
    Descrição, Classificação, Valor "R$ 1.234,56"). Stream via `streamDownload`.
  - **PDF:** `Pdf::loadView('exports.monthly-report')->download(...)` — view Blade própria com
    a paleta azul/verde do design system.
  - `GET /api/export/monthly?month=aaaa-mm&format=csv|pdf` (default csv).
- **Export completo LGPD (`DataExportService` + `ReportExportController@full`):** reúne perfil
  (sem senha/tokens) + todas as entidades do usuário (categorias, transações, orçamentos, metas,
  investimentos, recorrências), **incluindo soft-deletados** (`withTrashed`) para ser de fato
  completo. JSON em CENTAVOS, download via stream. `GET /api/export/full`.
- **Exclusão de conta LGPD (`AccountController@destroy`):** `DELETE /api/account` com
  **reautenticação por senha** (`current_password`). Política = **PURGE FÍSICO** (exceção
  explícita ao soft delete prevista no CLAUDE.md): apaga a linha do usuário (model User não usa
  soft delete) e o `cascadeOnDelete` das FKs remove fisicamente todas as entidades pessoais.

## Important Findings
- **Bug sutil corrigido (registrado para quem mexer em deleção de usuário):** `Auth::guard()->
  logout()` chama `cycleRememberToken()` que faz `save()` no usuário — se chamado **depois** do
  delete, **reinsere** a linha. Solução: deslogar/invalidar a sessão **antes** de `$user->delete()`,
  mantendo a referência ao model. Também evitei `DB::transaction` em volta do delete único (o
  cascade já é atômico e o savepoint conflitava com o RefreshDatabase nos testes).
- **Contratos JSON para a UI 5.4:**
  - `GET /api/export/monthly?month=aaaa-mm&format=csv|pdf` → download (`relatorio-aaaa-mm.csv|.pdf`).
    CSV: `text/csv; charset=UTF-8`, separador `;`, BOM, valores "R$ 1.234,56". Default format=csv.
  - `GET /api/export/full` → download JSON (`flowfin-dados-<id>.json`); **valores em CENTAVOS**;
    chaves `perfil, categorias, transacoes, orcamentos, metas, investimentos, recorrencias`.
  - `DELETE /api/account` body `{ "password": "<senha atual>" }` → 200 `{ message }`; 422 se a
    senha não confere. **Irreversível.**
- **Decisão a confirmar com o Manager:** política LGPD = **purge físico** (hard delete via
  cascade), não anonimização. Atende ao "direito ao esquecimento"; se o produto preferir
  anonimização (manter linhas agregadas anônimas), é ajuste no `AccountController`.
- **Para o usuário não-técnico (deploy):** nenhuma credencial externa nova; DomPDF é pura PHP.
  O export usa memória/CPU do request — ok para volumes pessoais.
