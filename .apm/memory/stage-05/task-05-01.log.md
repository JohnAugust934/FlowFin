# Task 5.1 - PWA Base

## Summary
FlowFin agora é um PWA instalável: manifest + ícone, Service Worker com cache da casca do app (interface abre offline) e prompt de instalação. Build compila e os 145 testes seguem verdes.

## Details
- **Manifest** (`public/manifest.webmanifest`): nome/short_name, descrição PT-BR, `display: standalone`, `start_url: /dashboard`, `scope: /`, `theme_color #2563EB` (azul da marca), `background_color #fff`, `lang pt-BR`. Ícone único SVG declarado com `purpose` `any` e `maskable`.
- **Ícone** (`public/img/pwa/app-icon.svg`): marca FlowFin em branco sobre fundo gradiente azul full-bleed (compatível com recorte maskable; mantém a marca na zona segura central). SVG escala para qualquer tamanho — ver decisão sobre PNG nos Important Findings.
- **Service Worker** (`public/sw.js`), estratégias por tipo:
  - Navegações (HTML): **network-first** — busca a versão fresca; sem rede, devolve a última navegação cacheada e, em último caso, `offline.html`. Evita servir HTML velho após deploy.
  - Assets do Vite (`/build/...`, com hash, imutáveis): **cache-first**.
  - Demais GET de mesma origem (ícones/manifest/fontes): **stale-while-revalidate**.
  - `/api/...`: **nunca cacheado** (dados financeiros; offline de escrita é a Task 5.2).
  - Cache versionado (`CACHE_VERSION = v1`); `activate` limpa caches antigos. `skipWaiting`/`clients.claim` para ativar rápido.
- **Registro + instalação** (`resources/js/flowfin/pwa.js`): registra `/sw.js` no evento `load` (não interfere no script anti-flash de tema do `<head>` nem no `@vite`). Captura `beforeinstallprompt`, expõe `window.FlowFin.pwa` e emite `flowfin:pwa-installable`.
- **Página offline** (`public/offline.html`): tela leiga PT-BR, autossuficiente (CSS inline, sem depender do bundle), recarrega sozinha ao voltar a conexão.
- **Casca/head** (`resources/views/layouts/app.blade.php`): `<link rel="manifest">`, `theme-color`, metas `apple-mobile-web-app-*` e `apple-touch-icon`.
- **Convite de instalação** (`resources/views/components/pwa-install.blade.php` + Alpine `pwaInstall`): banner discreto que só aparece quando o navegador sinaliza instalável; "Instalar" dispara o prompt nativo; "Agora não" é lembrado em `localStorage`.

## Important Findings
- **Ícone em SVG (sem PNG):** o ambiente não tem rasterizador de imagem (ImageMagick/Inkscape/sharp ausentes), então o ícone do PWA é um SVG (`sizes: any`), que o Chrome aceita como instalável em Android/desktop. **Limitação:** o iOS/Safari não usa SVG como `apple-touch-icon` — no iPhone o ícone da tela inicial pode cair num fallback. Se o alvo incluir iPhone com ícone perfeito, é preciso gerar PNGs 192/512/maskable (exige adicionar um rasterizador de build ou fornecer os PNGs prontos). Decisão tomada para não introduzir dependência nativa pesada; sinalizo para o Manager decidir.
- **start_url `/dashboard`** exige sessão; deslogado, redireciona para login (comportamento aceitável). O SW cacheia a navegação após a 1ª visita online.
